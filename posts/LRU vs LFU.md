# 核心区别

LRU 和 LFU 的核心区别在于**淘汰依据**不同：

- **LRU（最近最少使用）**：淘汰**最长时间未被访问**的数据。它关注“时效性”，认为最近没访问过的数据，未来也不太可能访问。
    
- **LFU（最不经常使用）**：淘汰**访问频率最低**的数据。它关注“热度”，认为只被访问过一次的数据不如被频繁访问的数据重要。
    

  

**具体实现与行为差异：**

- **LRU 实现**：通常用**哈希表 + 双向链表**。访问数据时将其移到链表头部，淘汰时移除链表尾部。
    
- **LFU 实现**：通常用**哈希表 + 频率链表**（如每个频率对应一个子链表）。访问时更新频率计数，淘汰时移除频率最低且最旧的数据。
    

  

这两者的区别是，

- **LRU** 是在对于访问的时间最远的那一个给他淘汰掉；而 **LFU** 是给所有缓存的数据都配一个**计数器**，访问一次就加1，然后对**访问计数最少**的淘汰掉。
    
- **LRU** 就是直接相当于给定一个顺序队列，访问一个出现在队列中的元素就给他重新插到队尾，每次淘汰队首元素；而对于 **LFU**，是对每个元素都有一个计数器，然后每次要淘汰的时候就遍历整个数组/列表，把计数器的数值最少的淘汰。
    

  

# **典型示例**

  

**缓存容量=3**，**访问顺序**：`A, A, A, B, C, D`

- **LRU 执行过程**：
    
    - 访问 A（3 次）、B、C 后，缓存为 `[A, B, C]`（A 因最近被反复访问，位于最近使用位）。
        
    - 访问 D 时，最久未使用的是 A（A 在 B 和 C 之后未被再访问）。
        
    - **淘汰 A**，新缓存为 `[B, C, D]`。
        
- **LFU 执行过程**：
    
    - A 访问 3 次（频次=3），B 和 C 各 1 次。缓存为 `{A:3, B:1, C:1}`。
        
    - 访问 D 时，频次最低的是 **B 和 C**（均为 1），按规则淘汰其中最旧的（假设是 B）。
        
    - **淘汰 B**，新缓存为 `{A:3, C:1, D:1}`。
        

  

- **LRU 结果**：访问 D 时，最久未用的是 B，因此淘汰 **B**。因为 A 刚被用过，虽然总次数多，但 B 已很久没被用。
    
- **LFU 结果**：访问 D 时，频率最低的是 B 和 C（各 1 次），按规则淘汰其中最旧的 **B**。若规则为淘汰最低频且最久未用，则同样是 B。
    
      
    

**真正的区别在**：LRU 需要“高频旧数据”，LFU “低频新数据”

  

**关键区别场景**：

当有**热点数据**时，差异显著。例如突然访问一次的数据：

- **LRU** 会淘汰该低频数据，保护高频热点。
    
- **LFU** 会保留该数据一段时间，可能导致高频热点因计数周期问题被误淘汰（需配合老化机制优化）。
    

  

# **知识扩展**

**LFU-Aging**：是 **LFU** 有一个常见变种，用来解决“历史高频但近期不再使用的数据”永远占着位置的问题。核心依然是纯 LFU（只计次数），但**定期把所有计数器的值除以 2**（或减去一个值）。这样，长期不用的“历史高频”数据，其计数值会随着时间衰减，变得比当前高频数据小，从而能被淘汰。

  

混合算法（如 **LRFU**）：同时计算时间和频率，给两者**分配权重**（例如 70% 看频率 + 30% 看时间）。这是一个真正的融合模型。
