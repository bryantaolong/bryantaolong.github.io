# Python

如果说让我推荐一门编程语言给一个**不熟悉编程**或者计算机的人的话，那一定是 Python，它的语法极为贴近自然语言，简单易懂，也没有过多的计算机底层细节，正如它的作者吉多·范罗苏姆（Guido van Rossum）所说，这是一门“**简洁易读，优雅明确**”的语言。

Python 的简练使其极为适合**快速验证**想法，这是**脚本语言**的精髓。

再者，Python 的**生态非常丰富**，不谦虚地说，其生态应当是所有编程语言的 Top 1。这是非常重要的，特别是对于一门脚本语言，丰富的生态对于一门脚本语言，意味着可以非常便利地做很多事情，意味着效率。

**缺点**
Python 的缺点也非常明显，动态鸭子类型、全局锁 GIL（Global Interpreter Lock）都是非常突出的问题。这当然不是说这不好，而是在某些领域，这些特征非常不适用。这是一个 trade off 的问题。

Python 的种种特征使其**定位**异常明确，一门**胶水**、**脚本**语言，它更像是一个玩具，一个瑞士军刀（可能不是很贴切），当你想利用计算机来干点什么**促进效率**的事情时，你会第一时间想到 Python，但是当你需要做一个产品，做一个工程项目时，它可能就没那么适用了。

# C Programming Language

而如果说你是一个对计算机有兴趣，想要了解计算机世界的人，我会推荐你学习 C 语言来入门，这不是说它是最好的语言，而是说，它非常适合人们了解计算机。以下是我的理由。

C 语言起初是贝尔实验室的 Unix 研究者们为了便利地编写操作系统而创造的语言。它有着很多与计算机硬件概念高度契合的设计，比如指针，内存管理等。同时，由于与计算机底层关系紧密，C 语言的性能也非常卓越，可以说就是简化了的汇编语言。

**缺点**
当然，任何一门编程语言都是有其局限的。**一个人最大的优点和最大的缺点可能来自于同一特质**，编程语言也不例外，C 语言的计算机底层友好性，也带来了可扩展性、易读性、工程化的问题。

# C++

C++ 的诞生源于**面向对象编程**思想，由面向对象思想与工程化的对撞，Bjarne Stroustrup 基于**零成本抽象**的理念，设计了 C plus plus programming language。

C++ 封装并简化了 C 语言的开发，在当时，很大程度上提升了开发者的开发效率。

# Java

Java 是 James Gosling 在为了简化 C++ 的一些问题，他最初是理念是“C++ --”。

Java 是一门非常**纯粹的面向对象**的语言，可以说是面向对象思想最极致的体现。这也为 Java 后来无比庞大的企业级应用生态奠定了基础。

# JavaScript

JavaScript 最初是为了给**浏览器**增加动态性，让网页动起来的一门脚本语言。浏览器是一款与用户极为接近的应用，在浏览器上，用户有很多**输入输出**操作，比如说点击、拖动等。因此 JavaScript 在设计之初就考虑到了这一点，其**事件驱动**与**异步模型**就是最好的体现。


JavaScript 是一门脚本语言，很容易想到它可能是一门动态语言。作为一门动态的解释型的脚本语言JavaScript 也有着很多 Python 有的优点， **Node.js** 将这些优点带出了浏览器，进入了更广泛的使用领域。不过 JavaScript 在代码的简洁性与可读性上与 Python 还是有着一定的差距，但这并不是说 JavaScript 就不如 Python，比如在 Web 领域，JavaScript 就是当之无愧的王者，因为所有浏览器都需要 JavaScript 来使页面动起来，都需要 JavaScript 来编写业务逻辑，来与后端对接。


由于需要进行前端的开发，JavaScript 在工程性上也比 Python 更具优势，这意味着（对比 Python）它更利于或者说他更适合于作用于大型项目的开发。


不过由于动态类型语言的固有缺陷，JavaScript 在一个非常大型的工程中还是有着亟需解决的问题。

## TypeScript

基于前端大型**工程化**的需要，TypeScript 应运而生。微软为 JavaScript 设计了一组类型集。作为 JavaScript 的超集，这就是 TypeScript。

# Go/Golang

Go，或者说 Golang 是 Google 为了解决现代的一些工程问题而设计的。其设计者包括了 Unix 的设计者之一——Ken Thompson。


Go 语言很好地**平衡了性能、可读性、工程化**这个编程语言不可能三角。其设计哲学是“**less is more**”，强调少即是多，某种程度上可以说是第一性原理的体现。

Go 语言是一门活在服务端的语言，它非常适合于用来搭建 infrastructure——基础设施，Kubenetes, Docker 将这一特点体现得淋漓尽致。
